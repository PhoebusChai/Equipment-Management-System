# 设备与实验室管理系统需求文档

## 1. 文档说明

- 项目名称：设备与实验室管理系统
- 目标用户：学生、教师、管理员
- 业务域：高校实验室与设备预约、审批、运维、分析
- 固定实验室业务范围：计算机实验室、生物实验室

## 2. 角色与权限

### 2.1 学生端

- 注册登录、维护个人信息
- 查询实验室与设备详情
- 发起预约（实验室/设备/紧急预约）
- 查询预约记录
- 提交设备评价、发起报修
- 查看系统通知

### 2.2 教师端

- 实验室申请、管理我的实验室
- 设备台账维护、设备申请、文档上传
- 审批学生预约
- 制定并上传定期设备保养计划
- 处理报修、维护设备/实验室状态
- 发布实验室公告/设备公告

### 2.3 管理员端

- 导入并管理学生/教师账号
- 审核实验室申请与设备申请
- 设备资产入库、维护记录管理、报废处理
- 全局数据分析（利用率、行为分析、耗材预警）

## 3. 业务功能需求

## 3.1 学生端功能模块

### 3.1.1 用户中心

- 个人信息管理：查看和编辑联系方式
- 信用积分：显示预约信用分
- 登录注册：支持邮箱注册登录

### 3.1.2 实验室与设备

- 实验室列表：按楼宇/学院分类搜索
- 设备目录：按关键字搜索设备
- 设备详情查询：
  - 技术参数/使用说明
  - 实时状态（可用/预约中/维修中）
  - 历史使用评价

### 3.1.3 预约功能

- 实验室/设备预约：
  - 按时间段选择（15 分钟粒度）
  - 冲突检测提醒
  - 紧急预约通道
  - 预约结果查询
- 预约记录：
  - 当前预约（含倒计时提醒）
  - 历史记录统计

### 3.1.4 消息中心

- 系统通知：实验室公告、设备公告、预约结果提醒

### 3.1.5 反馈评价

- 设备评价：使用体验反馈
- 报修申请：设备故障快速上报

## 3.2 教师端功能模块

### 3.2.1 实验室管理

- 实验室申请
- 我的实验室（负责列表）
- 设备台账：
  - 申请设备
  - 设备信息维护（状态/位置）
  - 使用文档上传
- 预约审核：学生预约审批流程
- 定期设备保养计划上传
- 设备申请与结果查询

### 3.2.2 预约监管

- 实时看板：当前使用情况可视化
- 预约日历：按日/周/月查看
- 冲突协调：处理冲突预约申请

### 3.2.3 使用监控

- 签到管理：异常签到处理
- 超时监控：超时提醒

### 3.2.4 设备/实验室报修

- 查看学生报修申请
- 更改设备/实验室状态（可用/维修中）
- 确认保修：报修确认后停止预约
- 恢复预约：报修结束后恢复资源预约
- 维修工单上交

### 3.2.5 设备报废申请

- 提出设备报废申请

### 3.2.6 公告发布

- 发布实验室公告/设备公告

## 3.3 管理员端功能模块

### 3.3.1 系统管理

- 用户管理：
  - 学生/教师账号导入
  - 查询全部用户信息
- 实验室管理：
  - 审核实验室申请
  - 查询实验室详情
- 设备管理：
  - 资产入库（采购信息记录）
  - 设备申请审核
  - 维护记录：
    - 查询保养计划
    - 维修工单审核与跟踪
  - 报废处理：审核设备报废申请

### 3.3.2 数据分析

- 利用率报表：设备/实验室使用频率
- 用户行为：高频使用人员分析
- 耗材补充预警：达到阈值自动提示

## 4. 核心业务流程

### 4.1 预约流程（业务完整）

1. 学生查询资源（实验室或设备）
2. 选择时间段（15 分钟粒度）
3. 系统执行冲突检测
4. 提交预约申请
5. 教师/系统审核（按策略）
6. 返回预约结果并通知
7. 使用签到与超时监控
8. 结束后支持评价与记录归档

### 4.2 报修流程

1. 学生提交报修
2. 教师确认报修并切换资源状态为维修中
3. 自动停止对应资源预约
4. 提交与跟踪维修工单
5. 维修完成后恢复预约

### 4.3 申请审核流程

涉及实验室申请、设备申请、设备报废申请：

1. 教师发起申请
2. 管理员审核
3. 结果通知
4. 写入资产/状态/审计记录

## 5. 非功能需求

- 可用性：核心业务 99.9% 可用
- 性能：列表查询接口 P95 小于 500ms
- 安全：
  - 密码加密存储
  - 角色权限控制（RBAC）
  - 关键操作审计日志
- 可维护性：模块化设计，前后端分离
- 可扩展性：支持新增实验室类型与预约策略

## 6. 前后端技术要求

### 6.1 前端

- 框架：Vue3 + JavaScript
- 样式：Tailwind CSS
- 图标：Lucide
- 请求：Axios
- 建议：按角色拆分路由与菜单，统一错误处理与拦截器

### 6.2 后端

- 框架：Spring Boot
- 分层：Controller / Service / Repository
- 能力：
  - JWT 会话鉴权（建议）
  - 预约冲突检测
  - 状态流转与审批流
  - 任务调度（超时提醒、保养提醒）

### 6.3 数据库

- MySQL 8.x
- 事务保证预约与状态一致性
- 建立时间段查询与状态查询索引

## 7. 验收标准（关键）

- 学生可完成注册登录、预约、查询记录、报修、评价全链路
- 教师可完成审批、监管、报修处理、公告发布
- 管理员可完成用户导入、申请审核、维护跟踪、报废处理与报表查看
- 预约冲突检测准确，15 分钟粒度生效
- 固定实验室（计算机/生物）业务可正常运行
