# 设备与实验室管理系统数据库设计（MySQL）

## 1. 设计目标

数据库设计需支持以下核心能力：

- 多角色用户体系（学生/教师/管理员）
- 固定实验室业务（计算机实验室、生物实验室）
- 预约三大范围（实验室预约、设备预约、紧急预约）
- 报修、保养、报废、公告、评价、签到、审计

## 2. 设计原则

- 主键统一使用 `BIGINT`（雪花 ID 或自增 ID）
- 所有表包含审计字段：`created_at`、`updated_at`
- 状态字段用枚举字符串或字典编码（建议字符串，便于可读）
- 核心查询建立复合索引（资源 + 时间 + 状态）
- 重要业务支持软删除（`is_deleted`）

## 3. 核心实体关系

- 用户 `users` 内置角色字段 `role_code`（STUDENT/TEACHER/ADMIN）
- 实验室 `labs` 与设备 `devices`：一对多
- 预约 `bookings` 关联用户和资源（实验室或设备），并内置冲突处理字段
- 评价 `device_reviews` 关联设备与预约
- 报修 `repair_requests` 关联资源与报修人
- 工单 `repair_work_orders` 关联报修单
- 公告 `notices` 支持实验室/设备维度
- 申请单（实验室/设备/报废）均含发起人与审核链

## 6. 关键 SQL 草案（简化）

```sql
CREATE TABLE users (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  role_code VARCHAR(32) NOT NULL,
  email VARCHAR(128) NOT NULL UNIQUE,
  password_hash VARCHAR(255) NOT NULL,
  real_name VARCHAR(64) NOT NULL,
  avatar_url VARCHAR(255),
  phone VARCHAR(32),
  credit_score INT NOT NULL DEFAULT 100,
  status VARCHAR(32) NOT NULL DEFAULT 'ACTIVE',
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

CREATE TABLE labs (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  lab_code VARCHAR(64) NOT NULL UNIQUE,
  lab_name VARCHAR(128) NOT NULL,
  lab_type VARCHAR(32) NOT NULL,
  building VARCHAR(64),
  college VARCHAR(128),
  capacity INT,
  manager_teacher_id BIGINT,
  status VARCHAR(32) NOT NULL DEFAULT 'AVAILABLE',
  description TEXT,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

CREATE TABLE devices (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  lab_id BIGINT NOT NULL,
  device_code VARCHAR(64) NOT NULL UNIQUE,
  device_name VARCHAR(128) NOT NULL,
  category VARCHAR(64),
  brand VARCHAR(64),
  model VARCHAR(64),
  technical_params TEXT,
  usage_guide_url VARCHAR(255),
  location VARCHAR(128),
  status VARCHAR(32) NOT NULL DEFAULT 'AVAILABLE',
  purchase_date DATE,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  CONSTRAINT fk_devices_lab FOREIGN KEY (lab_id) REFERENCES labs(id)
);

CREATE TABLE bookings (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  booking_no VARCHAR(64) NOT NULL UNIQUE,
  user_id BIGINT NOT NULL,
  resource_type VARCHAR(32) NOT NULL,
  resource_id BIGINT NOT NULL,
  booking_scope VARCHAR(32) NOT NULL,
  start_time DATETIME NOT NULL,
  end_time DATETIME NOT NULL,
  duration_minutes INT NOT NULL,
  status VARCHAR(32) NOT NULL DEFAULT 'PENDING',
  approver_id BIGINT,
  reject_reason VARCHAR(255),
  conflict_with_booking_id BIGINT,
  conflict_reason VARCHAR(255),
  conflict_resolved TINYINT(1) NOT NULL DEFAULT 0,
  conflict_resolved_by BIGINT,
  conflict_resolved_at DATETIME NULL,
  checkin_time DATETIME NULL,
  checkout_time DATETIME NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  CONSTRAINT fk_bookings_user FOREIGN KEY (user_id) REFERENCES users(id),
  CONSTRAINT fk_bookings_approver FOREIGN KEY (approver_id) REFERENCES users(id),
  CONSTRAINT fk_bookings_conflict_with_booking FOREIGN KEY (conflict_with_booking_id) REFERENCES bookings(id),
  CONSTRAINT fk_bookings_conflict_resolved_by FOREIGN KEY (conflict_resolved_by) REFERENCES users(id),
  INDEX idx_booking_resource_time (resource_type, resource_id, start_time, end_time, status),
  INDEX idx_booking_user (user_id, status, start_time)
);
```

## 7. 冲突检测规则（数据库层建议）

- 应用层判断：同一资源、时间区间重叠且状态属于有效态（PENDING/APPROVED/IN_USE）视为冲突。
- SQL 条件（示意）：
  - `new_start < existed_end AND new_end > existed_start`
- 建议在事务中执行：
  1. 查询冲突
  2. 插入预约
  3. 更新资源状态或触发审批
